# 무한 파르페 정책 설계 (G-001)

- **원본**: `raw/G-001-무한파르페-정책설계.md` (+ `raw/assets/G-001-*.png` 2장)
- **날짜**: 2026-06-13 (시뮬레이션 캡처 기준)
- **도메인**: research
- **성격**: [[기능정의서-v5]]의 G-001 행을 화면 단위로 상세화한 정책 설계서. 기능정의서 시리즈와 별개 소스.
- **상태**: 🔁 **대체됨(superseded)** — **배치·그리드·토핑 모델**(2-1-2-1 스태거·index%3·프레임 규격)은 [[G-001-무한파르페-간격-정책-v0.2]](지그재그·160 프레임·6타입)로 대체. 조회·정렬(활동순)·상태별·라벨·상대시간 등 **운영 스펙**은 [[무한-파르페-그리드]] 개념이 현행으로 보유.

## 핵심 요점
- **화면**: G-001 그룹 목록 메인. 그룹이 늘수록 파르페 템플릿이 아래로 무한히 길어짐.
- **조회**: 한 번의 API 호출로 전체 조회(페이지네이션 없음). 필드 = 그룹ID·프리뷰이미지·이름·활동시간.
- **정렬**: 가입순/활동순/이름순 투표 → **활동순** 다수 선택(자주 보는 그룹이 위로).
- **배치**: [[무한-파르페-그리드|2-1-2-1 스태거 패턴]]. index % 3 규칙(0·1 → 2열 좌우, 2 → 1열 full-width). N=0 빈 상태, N=1 가운데.
- **파르페 요소**: 크림(배경 패딩)·접시(기능 없음)·체리(TBD)·[[토핑]](대표 프리뷰, 클릭 시 C-001)·라벨(이름 칩 + 상대시간).
- **라벨**: 이름 최대 6자, 7자+ → 앞 5자 + `…`. 상대시간 5단계(방금 전/N분·N시간·N일 전/오래 전). NEW = C-001 진입 시 해제, 폴백 `lastActivityAt` 24h 이내.
- **상태별**: 로딩(스켈레톤)/성공/0건(Empty)/실패(Error+새로고침).
- **갱신**: Pull-to-Refresh 전체 교체 + 스크롤 최상단 초기화. 그룹 나가기 시 뒤 토핑 한 칸씩 당김 → 패턴 전체 재계산(저장·재진입 시 반영, 실시간 아님).
- **캐싱**: 프리뷰 이미지 지연 로딩 + 캐싱. 그룹 캐싱 여부는 보류(취소선).
- **z-index**: 접시 1 < 크림 2 < 체리 3 < 토핑/라벨 4.
- **시뮬레이션**: 피그마 site 제공(<https://model-turn-68745673.figma.site/>). 토핑 클릭은 실제 C-001 이동이나 시뮬은 그룹 삭제 바텀시트로 대체.

## 미결 (→ [[open-questions]])
- 토핑 이미지 뷰어 정책 미정("다음에 정함").
- 크림 가로 길이 1안(그리드 전체)/2안(풀행 중앙 정렬) 미정.
- 정렬 기준은 투표 결과일 뿐 최종 확정 문구 아님.
