# 무한 파르페 정책 설계서 v0.2 (G-001)

- **원본**: `raw/G-001-무한파르페-정책설계-v0.2.md`
- **날짜**: 2026-07-22 기준 (v0.2)
- **도메인**: research
- **성격**: [[G-001-무한파르페-정책설계]](v0.1, 2026-06-13)의 **후속 정본**. 배치 모델(지그재그)뿐 아니라 조회·상태·갱신·라벨 등 **운영 스펙 전반을 v0.2로 확정**. 배치 상수 상세는 [[G-001-무한파르페-간격-정책-v0.2]]가 보유(동일 v0.2 값).
- **상태**: ✅ **현행 정본** — 이 소스가 G-001 정책 설계서의 최신본. v0.1은 대체됨.

## 핵심 요점 (v0.1 → v0.2 변경 중심)
- **배치**: 2-1-2-1 스태거(index%3) → **지그재그**(행당 1개, 좌/우, index%2). 첫 토핑 Left 고정. → [[무한-파르페-그리드]] / [[G-001-무한파르페-간격-정책-v0.2]].
- **정렬**: 투표 중 → **활동순 확정**. 단일 API 호출, 페이지네이션 없음.
- **상태별**: 초기 로딩 = **자체 로딩 그래픽**(인디케이터·스켈레톤 대신 제작, 화면 동작 불가) / 성공 / 0건 = **온보딩 툴팁(G-001-Empty)** 담당·상시 노출 / 실패 = Error + 새로고침.
- **갱신**: Pull-to-Refresh 전체 교체·스크롤 최상단 초기화. 재진입 시 **자동 재조회 확정**.
- **그룹 나가기**: 뒤 토핑 한 칸씩 당김 → side=`index%2`라 **당김 지점 이후 좌/우·타입·회전이 바뀜(정상 동작)**. 변형 1/2/3 시드는 그룹ID 고정이라 유지. 저장·재진입 시 반영(실시간 아님).
- **캐싱**: 프리뷰 이미지 지연 로딩 + 캐싱.
- **토핑**: 이미지 뷰어 정책 **삭제 확정**(불필요). 고정 사이즈(프레임 160×160 / Img 96×96). 클릭 시 C-001 이동.
- **라벨/그룹명**: **80px 초과 시 초과분부터 `…`(픽셀 기준)**. 구 "6자/7자→앞5자+…" 문자수 기준 **폐기**. 시간 텍스트는 항상 전체 노출. Timestamp 컬러 = [[S-101-프로필-닉네임-컬러-규칙-v0.3]] 준수.
- **상대 시간**: 방금 전(60초 미만)/N분/N시간/N일/오래 전(7일+). (v0.1과 동일)
- **NEW 기준**: 그룹을 C-001로 선택해야 읽음 처리. 화면 노출만으론 해제 안 됨.
- **크림**: 개수 규칙 — 토핑 0~3개 크림 3개, **4개부터 토핑 1개당 크림 1칸 추가**.
- **저개수 배치**: N=1/2 → N≥3 대비 각 토핑 **y +12 내림**(x 동일). N=3 스크롤 없음. N≥4 스크롤·크림 성장.
- **오버플로우 없음**: 96 Img는 160 프레임 회전·이동해도 벗어나지 않음 → 구 clip OFF/삐짐 노출 정책 삭제.
- **z-index**: 접시1 < 크림2 < 체리3 < 토핑/라벨4. 토핑 내부 Img > Grouptag-Chip.
