# Nametag-Chip (프로필·닉네임 컬러 규칙)

[[협업-캔버스-앱]]에서 유저를 표현하는 **닉네임 첫 글자가 들어간 원형 프로필 이미지**. 유저 타입(12종)이 여러 UI 요소의 텍스트 컬러를 강제 결정하는 기준. S-101(그룹 사이드 메뉴)에서 다른 유저를 볼 때 노출. **타입→컬러 매핑은 요소별로 별개**다: 토스트 닉네임([[S-101-프로필-닉네임-컬러-규칙-v0.3]])과 Grouptag-Chip Timestamp([[S-101-프로필-이미지-및-그룹칩-내-타임스탬프-컬러-규칙-v0.1]])는 Cherry 계열 단계가 서로 다름(같은 값이어야 하는 규칙 아님).

## 구성
- **6개 컬러**(연핑크·진핑크·체리·그레이·멜론·푸딩) × **각 2타입** = **총 12종**.
- 계정당 **12종 중 1종 랜덤** 배정.

> ⚠️ 2026-07-18 TJYG-Android 구현(`YGColorChipType`)과 상충 — 코드는 `NametagChip1`~`NametagChip13`(13종) + `NametagChipPlus`(추가용)로 정의돼 **정책 12종과 개수 불일치**. → [[open-questions]] 등록, parfait `open-questions`([2026-07-18])에도 교차 기록.

## 컬러 매핑 (요소별 별개 — 상충 아님)

타입→컬러 매핑은 **UI 요소마다 별개 표**를 쓴다. 그레이/멜론/푸딩은 두 표 동일, **Cherry 계열만 단계가 다름**.

### ① 토스트 닉네임 텍스트 ([[S-101-프로필-닉네임-컬러-규칙-v0.3]])
공용 캔버스 액션 발생 시 동접 그룹원에게 뜨는 토스트의 **닉네임 텍스트** 컬러.

| 컬러 (타입) | 텍스트 컬러 |
|---|---|
| 연핑크 (Type 1, 2) | Cherry-200 |
| 진핑크 (Type 3, 4) | Cherry-300 |
| 체리 (Type 5, 6) | Cherry-400 |
| 그레이 (Type 7, 8) | White |
| 멜론 (Type 9, 10) | Melon |
| 푸딩 (Type 11, 12) | Pudding |

### ② Grouptag-Chip Timestamp 텍스트 ([[S-101-프로필-이미지-및-그룹칩-내-타임스탬프-컬러-규칙-v0.1]])
G-001에서 그룹 대표 토핑과 노출되는 Grouptag-Chip의 **Timestamp 텍스트** 컬러. 기준 = 해당 그룹에서 **마지막으로 변화를 가한**(토핑 추가·편집, 배경 추가·변경) 유저의 타입.

| 컬러 (타입) | 텍스트 컬러 |
|---|---|
| 연핑크 (Type 1, 2) | Cherry-100 |
| 진핑크 (Type 3, 4) | Cherry-200 |
| 체리 (Type 5, 6) | Cherry-300 |
| 그레이 (Type 7, 8) | White |
| 멜론 (Type 9, 10) | Melon |
| 푸딩 (Type 11, 12) | Pudding |

## 타입 고정 정책 (v0.2 추가, v0.3 유지)
- 타입은 **계정 생성 시 1회 랜덤 배정**, 이후 변경 없음.
- **동일 닉네임이라도 계정이 다르면 서로 다른 타입** 가능.
- 배정 타입은 모든 화면에서 동일 유지, 앱 재접속·그룹 변경에도 불변. 별도 타입 변경 기능 없는 한 계정 유지 동안 동일 타입.
- 예: 계정 A(닉네임 `홍씨`) → Type 1 고정, 계정 B(닉네임 `홍씨`) → Type 8 고정. 닉네임 같아도 계정 다르면 컬러 다름.
