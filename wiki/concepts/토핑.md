# 토핑

[[협업-캔버스-앱]]에서 캔버스에 배치하는 **누끼 딴 사진 객체**. 파르페(디저트) 위에 토핑을 얹는 메타포.

## 명칭 변천
- [[기능정의서-v2]]·[[기능정의서-v3]]: "누끼 사진" / "사진".
- [[기능정의서-v4]]부터 **"토핑"**으로 명칭 통일 (C-305 "토핑 편집", C-306 "토핑 테두리 편집").

## 라이프사이클
1. 촬영(C-101) 또는 갤러리(C-102)로 원본 사진 확보.
2. [[누끼-따기]](C-103)로 배경 제거 → 투명 PNG.
3. 수동 편집: 누끼 영역(C-104), 테두리·크기(C-105/C-306).
4. 캔버스 배치(C-106): 좌표(x, y, z-index), 크기. 확정 시 서버 저장 + 그룹 브로드캐스트.
5. 배치 후 수정: 파르페 편집 모드(C-301) → 토핑 편집(C-305, 자기 것만).

## 편집 정책
- MVP 편집 항목 = **크기, 테두리** 2가지로 고정.
- 자신이 올린 토핑만 위치/편집 가능.
- z-order·회전 허용 범위 미정([[open-questions]]).

## 탭 상호작용 (C-202, [[토핑-spotlight]])
- **본인 토핑 탭** → C-305 토핑 편집 진입.
- **타인 토핑 탭** → [[토핑-spotlight|Spotlight]] 상태(선택 토핑 강조 + 나머지 Dim) + 작성자 [[toast|Toast]] 1회. 상세는 [[토핑-spotlight]].

## 반환 데이터
- 스티커(토핑) 아이디, 유저 아이디, 좌표(x, y, z).

## 대표 토핑 (G-001)
- 그룹 목록에서 각 그룹은 **대표(최신) 토핑** 프리뷰 1개로 표현됨 ([[무한-파르페-그리드]]).
- 프리뷰 이미지 크기는 디자인 고정값(가변 아님), 지연 로딩 + 캐싱.
- 클릭 시 개별 토핑을 잡고 있는 **C-001**로 이동.
- 우측 하단 라벨(이름 칩 + 상대시간), NEW 표시는 [[무한-파르페-그리드]] 라벨 정책 참고.

## 토핑 컴포넌트 규격 (G-001, [[G-001-토핑-정책-v0.1]])
G-001 그리드에 얹히는 토핑 단위 컴포넌트의 배치·변형 스펙 (v0.1, 2026-07-09).

- 구성 = **Img + Grouptag-Chip**, 3타입 (`1` / `2-Left` / `2-Right`).
- 기본 규격: 프레임 **180×180**, Img **144×144**(정중앙), Chip 회전 없음, Z는 **Img > Chip** 항상.
  - Chip Timestamp 텍스트 컬러 = [[nametag-chip]] 컬러 규칙 준수 (타입별 강제 매핑, 랜덤 X).
- 타입별 고정 상수 (정중앙 오프셋 0 기준):

| 타입 | Img Translate | Img Rotate | Chip Offset |
|---|---|---|---|
| 1 (1열) | X -10, Y 10 | -6° | -9 |
| 2-Left (2열 좌) | X 14, Y -6 | -4° | -6 |
| 2-Right (2열 우) | X 8, Y -12 | 8° | -12 |

- 반복: [[무한-파르페-그리드]]의 2-1-2-1 위에서 `1` → `2-Left`/`2-Right` → `1` … 순환.
- **오버플로우 = 안 자름** (Clip Contents OFF): 144 Img를 회전·이동하면 코너가 180 프레임을 넘어가는데, 자르지 않고 그대로 노출. 삐져나온 픽셀은 인접 토핑을 밀지 않으므로 자연스러운 분위기 유지. 프레임 경계 클립은 오답.
