# 화면 ID 체계

[[협업-캔버스-앱]] 기능정의서의 화면 식별 규칙. 접두사로 도메인 구분.

| 접두사 | 도메인 | 예 |
|---|---|---|
| `A-` | Auth (인증/온보딩) | A-001 스플래시, A-002 로그인 |
| `G-` | Group (그룹) | G-001 그룹 목록, G-002 그룹 진입 |
| `C-` | Canvas (캔버스/편집) | C-001 메인, C-101 촬영, C-301 파르페 편집 |
| `S-` | Sidebar (사이드 메뉴) | S-001 앱 메뉴, S-101 그룹 메뉴 |
| `SY-` | System (스케줄/자동) | SY-001 마감, SY-003 신규 생성 |

## 서브 ID (v5~)
[[기능정의서-v5]]에서 확인/에러/로딩 화면에 하이픈 서브 ID 도입:
- `C-101-confirm`, `C-101-error`, `C-102-confirm`, `C-102-error`
- `C-103-loading`, `C-103-select`
- `SY-001-closed`, `SY-001-new`

## 변화
- A-003의 의미가 버전에 따라 바뀜: v2~v4 "진입 경로 선택" → v5 "약관 동의"(→ [[open-questions]] 흐름 불일치).
- 편집 화면(C-1xx/C-3xx) 번호는 v3·v5에서 재배치됨.
