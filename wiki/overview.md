# Overview — 위키 논지

> 위키에 쌓인 지식 전체를 관통하는 살아있는 개요. 새 소스가 ingest될 때마다 갱신된다.

## 현재 논지 (thesis)

이 위키는 [[협업-캔버스-앱]]의 기획이 **기능정의서 진화(무엇을) → 화면 단위 컴포넌트 정책 확정(어떻게)** 으로 넘어가는 과정을 추적한다. 그룹 멤버들이 매일 자동 생성되는 공유 캔버스에 [[토핑]](누끼 딴 사진)을 올려 함께 꾸미는 모바일 앱이다. 기능정의서는 2차~5차(2026-05-30 ~ 06-25)에 걸쳐 MVP 범위를 좁히고 편집 흐름을 다듬었고, 이어 2026-07 초 **개별 화면(A-005·S-002·S-102·S-101·C-001·C-201·G-001)의 구현 직전 정책**이 v0.1로 쏟아지며 픽셀·상수·유효성 규칙까지 확정되는 단계에 진입했다.

> 구현 문서(ADR·architecture·specs·plans)는 별도 루트 `parfait/`로 분리(위키 스키마 밖). 이 위키는 **제품 정책 지식**만 담는다.

## 핵심 발견

- **MVP 축소 기조**: 그룹명/인원수 변경, 그룹장 기능 등 부가 기능을 버전이 올라갈수록 삭제. 편집 항목도 크기·테두리 2가지로 고정.
- **편집 구조 개편(v3)**: 편집 허브 C-002를 없애고 "파르페 편집 모드(C-301)"로 재구성. 최초 배치와 사후 수정 경로를 분리.
- **명칭 정리(v4)**: [[누끼-따기|누끼]] 결과물을 **[[토핑]]**으로 통일. 파르페 메타포 강화.
- **일 단위 캔버스 리듬(v4)**: [[캔버스-마감-스케줄]]의 하루 경계를 00시 → **03시**로 이동(갤러리 로드 범위도 동일 정렬).
- **개발 착수 신호(v5)**: 담당자(디자인/iOS/AOS/Spring) 배정, 에러·확인·로딩 서브 화면 보강 → 실제 구현 준비 단계로 진입.
- **화면 단위 상세화 시작(G-001)**: 기능정의서 표 1행이던 G-001이 독립 정책서로 확장 — [[무한-파르페-그리드]](파르페 요소·z-index), 정렬 활동순, 상대시간·NEW·라벨 글자수 정책 확정. 메타포가 편집 화면을 넘어 목록 화면까지 확장됨.
- **G-001 배치 모델 재설계(v0.2, 2026-07)**: 초기 `2-1-2-1` 스태거(index%3·180 프레임·3타입)를 **지그재그**(행당 1개 좌/우 교차·index%2·160 프레임·6타입 Left/Right-1/2/3)로 전면 재설계 → [[G-001-무한파르페-간격-정책-v0.2]]. [[G-001-무한파르페-정책설계]]·v0.1 배치 모델은 대체됨. (raw 토핑-정책-v0.1은 아직 구값 → [[open-questions]] 상충.)
- **컴포넌트 정책 v0.1 대량 확정(2026-07)**: 화면별 구현 직전 스펙이 픽셀·상수 단위로 확정.
    - [[토핑]] 컴포넌트(v0.2 현행: 160 프레임·96 Img·6타입 Translate/Rotate 상수·오버플로우 노출).
    - [[nametag-chip]]: 닉네임 첫 글자 원형 프로필 12종, 계정 생성 시 1회 랜덤 고정. 타입→컬러 매핑은 **요소별 별개** — 토스트 닉네임(Cherry-200/300/400)과 Grouptag-Chip Timestamp(Cherry-100/200/300, v0.3에서 분리된 별도 규칙)는 Cherry 단계가 다름(상충 아님).
    - [[이름-입력-규칙]]: 그룹명·닉네임 공통 입력 유효성(허용 문자·공백 규칙) + 화면별 상한(그룹명 10자 / 닉네임 15자, 앱·그룹 닉네임 동일).
    - [[캔버스-반응형-레이아웃]](C-001): 16:9 유지·최소 gap 12px·비례 스케일 + Background Blur 별도 레이어.
    - [[캘린더-컴포넌트]](C-201): Button-Date 4상태·Chip-Indicator·List-Date, Disabled(미래 날짜)→토핑 False.
    - [[토핑-spotlight]](C-202): 타인 [[토핑]] 탭 → Spotlight(선택만 강조 + 나머지 Dim Black-50, z: Spotlight>Dim>토핑>배경) + 작성자 [[toast|Toast]] 1회. 본인 토핑 → C-305 편집. Spotlight 직접 전환 불가(Default 경유). 탈퇴 유저 '알 수 없음'.
    - [[toast]] 공통 정책: 위→아래 노출, 스와이프업 or 2초 닫기, 나중 Toast를 이전 위로 스택. 닉네임 텍스트 컬러는 [[nametag-chip]] 매핑 준수.

## 열린 질문

- **A-003 흐름 불일치**: v5에서 A-003이 "약관 동의"로 바뀌었으나 A-004/A-005가 여전히 A-003을 진입경로로 참조. "그룹 참여/생성 선택" 화면 행방 불명.
- **G-002 캡처 누락**: v4·v5 캡처에 그룹 진입 화면 미포착.
- **다수 정책 미결**: 실시간 반영(WS vs 폴링), 누끼 온디바이스 vs 서버, 동시 편집 충돌 처리, 과거 캔버스 조회 기간 등. (닉네임 정책은 글자수·유효성 확정 → [[이름-입력-규칙]]; **금칙어·중복만 미결**로 좁혀짐.)
