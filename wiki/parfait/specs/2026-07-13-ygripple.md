# Spec: ygRipple — YG 커스텀 dim ripple IndicationNodeFactory

- 상태: 구현 예정
- 날짜: 2026-07-13
- 대상: `core:designsystem` — `ripple/` (신규 패키지)
- 관련: [ADR-0010](../adr/0010-custom-compositionlocal-theme.md)(자체 테마) · [design-system](../architecture/design-system.md) · [[2026-07-12-clickableyg-throttle|clickableYG]](이 ripple을 indication으로 받음)

## 목표
YG 자체 dim ripple을 `androidx.compose.foundation.IndicationNodeFactory`로 제공한다. `clickableYG`·`clickable`의 `indication` 인자로 끼워 은은한(subtle) 터치 리플을 낸다. 커스텀 테마(MaterialTheme 미사용)라 material3 기본 ripple 대신 색·alpha를 직접 제어한다.

## 범위
- **포함**: `ygRipple(...)` 팩토리 함수 + 내부 `IndicationNodeFactory`/delegating ripple `Node` + `YGDimRippleAlpha` 상수. 단일 파일.
- **제외**: designsystem 레벨 clickable wrapper(indication 기본값 주입) — 후속. `ygScaleRipple`(press 축소 효과) — 후속. `ColorProducer` 동적 색 오버로드(YAGNI). ripple 색의 시맨틱 토큰화(과도기, 후속).

## API / 인터페이스
```kotlin
@Stable
fun ygRipple(
    bounded: Boolean = true,
    radius: Dp = Dp.Unspecified,
    color: Color = YGAtomicColors.Gray.Gray900,
    rippleAlpha: RippleAlpha = YGDimRippleAlpha,
): IndicationNodeFactory
```
- `bounded`: 리플을 컨텐츠 경계로 클립할지. 기본 `true`.
- `radius`: 리플 반경. `Dp.Unspecified`면 컨텐츠 크기 기반 자동.
- `color`: 리플 색. 기본 `YGAtomicColors.Gray.Gray900`(어두운 회색; dim alpha라 subtle). 오버라이드 가능.
- `rippleAlpha`: 상태별 alpha. 기본 `YGDimRippleAlpha`.

## 동작 / 구조
- `ygRipple(...)`가 내부 `YGRippleNodeFactory`(= `IndicationNodeFactory`)를 반환.
- **`YGRippleNodeFactory`**: `create(interactionSource): DelegatableNode`가 `DelegatingYGRippleNode`를 생성. `equals`/`hashCode`를 파라미터(bounded/radius/color/rippleAlpha) 기반으로 구현 — Indication 동등성 판정에 필요.
- **`DelegatingYGRippleNode`**(`DelegatingNode`, `CompositionLocalConsumerModifierNode`): `onAttach`에서 `delegate(createRippleModifierNode(interactionSource, bounded, radius, color = ColorProducer { color }, rippleAlpha = { rippleAlpha }))`로 실제 리플 노드에 위임.
- **`YGDimRippleAlpha`**: `RippleAlpha(pressedAlpha = 0.15f, focusedAlpha = 0.15f, draggedAlpha = 0.15f, hoveredAlpha = 0.15f)`. 전 상태 균일 dim.
- `createRippleModifierNode`·`RippleAlpha`는 `androidx.compose.material.ripple`.

## 파일 구성 (`core:designsystem`)
- `ripple/YGRipple.kt` (신규 패키지 `com.teamyg.parfait.core.designsystem.ripple`) — public `ygRipple(...)` + `YGDimRippleAlpha` + `internal class YGRippleNodeFactory` + `private class DelegatingYGRippleNode`. 단일 파일.

## 주의 / 열린 질문
- **의존성**: `createRippleModifierNode`/`RippleAlpha`는 `androidx.compose.material:material-ripple`. material3 전이 포함으로 추정하되, 미해결 시 `material-ripple` 명시 의존 추가(plan compile에서 확정).
- **과도기 색**: ripple 색이 시맨틱 슬롯 아닌 `YGAtomicColors.Gray.Gray900` 직접 참조(YGButton·YGIconButton 선례). 디자인 토큰 규칙 확정 시 `effect/ripple` 시맨틱 토큰으로 정리 → [open-questions 2026-07-10 YGButton 디자인 토큰](../../synthesis/open-questions.md).
- **테마색 미읽음**: `ygRipple`은 색을 파라미터로 받으므로 테마 CompositionLocal을 직접 읽지 않음(호출측이 원하는 색 전달). 테마색 기본값 주입은 designsystem clickable wrapper 후속 과제.
