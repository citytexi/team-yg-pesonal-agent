# 토핑 편집자 확인 규칙

- **화면/컴포넌트**: C-202
- **적용 대상**: 1. Topping-Canvas, 2. Toast
- **날짜**: 26.07.14
- **버전**: v0.1
- **작성자**: DSN-B
- **개정이력**: v0.1 초안 작성

> 🔒 **실명 마스킹됨**: public repo이므로 작성자 실명을 역할 코드(DSN-B)로 마스킹함.

## 1. 개요

캔버스에 배치된 타인의 토핑을 탭하면 해당 토핑이 Spotlight 상태로 전환된다.

Spotlighted 상태에서는 선택한 토핑만 강조되고, 나머지 캔버스 영역은 Dim 처리된다. 동시에 작성자 정보가 Toast로 노출된다.

**본인 토핑을 탭한 경우에는 본 정책이 적용되지 않으며, C-305 토핑 편집으로 진입**한다.

## 2. 상태

### 1. Default
캔버스의 기본 상태이다. 모든 토핑이 동일한 레벨로 노출된다.

### 2. Spotlighted
선택한 토핑 1개만 강조되며, 나머지 캔버스 영역에는 Dim 레이어가 적용된다. Spotlighted 진입과 동시에 작성자 정보 Toast가 1회 노출된다.

## 3. 상태 전이

### 1. Default → Spotlighted
**타인의 토핑을 탭하면 Spotlight 상태로 전환**된다.

### 2. Spotlighted → Default
**Spotlight된 토핑을 제외한 캔버스 영역을 탭하면 Default 상태로 복귀**한다. Dim 영역을 탭하는 경우도 동일하게 처리한다.

### 3. Spotlighted → Spotlighted
Spotlight 상태에서 다른 토핑으로 바로 전환하는 기능은 지원하지 않는다. **다른 토핑을 선택하려면 반드시 Default 상태를 거쳐야 한다.** <!-- 원본 디자인 주석(빨강): "안거쳐도되게개발해줄거면댓글주세요 (환영)" -->

## 4. Spotlight 렌더링

### 1. Dim 레이어
Spotlight 상태에서는 선택된 토핑을 제외한 캔버스 전체 영역에 Transparency/Black-50 Dim 레이어를 적용한다. 배경과 다른 모든 토핑은 Dim 처리되고, Spotlight된 토핑만 영향을 받지 않는다.

### 2. Z-Index
Spotlight된 토핑은 캔버스 내 최상위 z-index로 이동한다. Dim 레이어는 Spotlight 토핑 바로 아래에 위치하며, 나머지 토핑은 기존 z-index를 유지한다.

우선순위는 다음과 같다.

Spotlight 토핑 → Dim 레이어 → 나머지 토핑 → 배경

## 5. Toast 정책

Spotlight 상태 진입 시 Toast를 1회 노출한다.

Toast 문구는 `{닉네임} 님이 {상대시간} 전에 쌓았어요` 형식을 사용한다.

Toast의 노출 시간 및 사라지는 방식은 **Toast 공통 규칙**을 따른다.

## 6. 예외 및 엣지 케이스

- 본인 토핑을 탭한 경우에는 Spotlight를 적용하지 않고 C-305 토핑 편집으로 진입한다.
- 탈퇴하거나 그룹을 나간 사용자의 토핑도 Spotlight는 정상 동작한다. 단, Toast에는 '알 수 없음'을 표시한다.
  - 예: `알 수 없는 사용자가 59분 전에 쌓았어요`, `알 수 없는 사용자가 오래 전에 쌓았어요`
- Spotlight 상태에서 앱이 백그라운드로 이동했다가 복귀하면 Spotlight를 해제하고 Default 상태로 복귀한다.
- Spotlight 상태에서 Pull-to-Refresh를 수행하는 경우에도 먼저 Default 상태로 복귀한 뒤 새로고침을 진행한다.
