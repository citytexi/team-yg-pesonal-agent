# 5. 갱신(새로고침) 정책

- Pull-to-Refresh로 전체 목록 다시 조회
- 새로고침 시 기존 리스트를 전체 교체하며 스크롤 위치는 최상단으로 초기화
- 화면 재진입 시 최신 활동 시간 반영을 위해 **자동 재조회 진행 (확정)**

## 5.1 그룹 나가기 정책

- 그룹 나가기→ 해당 토핑이 목록에서 빠짐.
- **빈자리 없음:** 나간 토핑 위치 뒤의 토핑들이 정렬 순서 그대로 한 칸씩 위로 당겨짐.
- 지그재그 재배정: side는 `index % 2`라, 한 칸 당겨지면 **당겨진 지점 이후 모든 토핑의 좌/우(및 타입·회전)가 바뀜**→ 버그 아니라 정상 동작. (변형 1/2/3 시드는 그룹 ID 고정이라 유지)
- 크림은 그만큼 수축, 접시가 위로 올라옴.
- **반영 시점:** 저장·화면 재진입 시 (실시간 아님 — 기존 동기화 정책).

# 8. 그리드 배치 정책

## 8.1 배치 패턴

- 리스트는 **지그재그**(행당 1개, 좌/우 번갈아) 패턴으로 반복 배치
- 위에서 아래로 **좌 → 우 → 좌 → 우** 무한 반복
- 첫 번째 토핑(index 0)은 **Left** 고정으로 시작

<aside>
💡

[ Left ]
         [ Right ]
[ Left ]
         [ Right ]
[ Left ]
...

</aside>

### 저개수 특수 배치

- **N = 0** → 토핑 없음. 크림 기본 (3개) + 접시. **안내는 온보딩 툴팁(G-001-Empty)이 담당**
- **N = 1 / N = 2** → 여백 고려, N ≥ 3 배치 대비 **각 토핑 y를 +12 내림(아래로)**. x는 동일.
- **N = 3** → 스크롤 없음, 파르페 세로 성장 없음.
- **N = 4 이상** → 스크롤 및 파르페 템플릿 크림 추가(세로 성장) 시작. 토핑 한개 늘 때마다 크림도 한개씩 늘어남.

## 8.2 인덱스 기반 규칙

아이템 순서를 0부터 셀 때, 2개 단위 반복 (좌/우 번갈아)

| 그룹 내 위치 (index % 2) | side |
| --- | --- |
| 0 | Left |
| 1 | Right |

### 좌표 및 세로 갭

- **기준 x (고정):** Left = 좌측 인셋 **28** (x=28) / Right = 우측 인셋 **28**
- **첫 토핑(Left, index 0): y = 280 // 저개수 특수 배치로 280에 위치함. 4개 이상부터의 첫 토핑은 268에 위치**
- **같은 side 세로 갭 = -12**
    - Left 열: y = **268, 416, 564, …**
- **Right 토핑은 인접한 두 Left 토핑의 가운데 높이**에 배치
    - Right 열: y = **354, 502, 650…**
- 변형 번호(1/2/3)는 세로 갭에 영향 없음 — 같은 side면 -12 동일

### 템플릿 고정 갭

| 구간 | 갭 |
| --- | --- |
| 체리 ↔ 첫 토핑 | -65 |
| 마지막 토핑 ↔ 접시 | -48 |
| 접시 ↔ 스크롤 인디케이터 | 최소 -16 (**스크롤 발생 시에만** 적용) |

### 토핑 규격 · 타입별 변형

토핑 = **Img + Grouptag-Chip**, **6타입** (`Left-1/2/3`, `Right-1/2/3`)

- 토핑 프레임: **160×160**
- Img: **96×96**
- **오버플로우 없음:** 96 Img는 160 프레임을 회전·이동(풀블리드 포함)해도 **절대 벗어나지 않음** → 구 clip OFF/삐짐 노출 정책 **삭제**
- Chip: 회전 없음. Timestamp 텍스트 컬러 = `프로필 이미지 및 토스트 닉네임 텍스트 컬러 규칙` 준수 (랜덤 X)
- Z: **Img > Grouptag-Chip** (항상)

| Type | Image Translate X | Image Translate Y | Image Rotate | Chip Offset |
| --- | --- | --- | --- | --- |
| Left-1 | 26 | 16 | 6° | -7 |
| Left-2 | 24 | 11 | 12° | -5 |
| Left-3 | 25 | 16 | -8° | -8 |
| Right-1 | 26 | 16 | -6° | -6 |
| Right-2 | 22 | 8 | -16° | -3 |
| Right-3 | 25 | 16 | -8° | -8 |

## 9.3 라벨

### 그룹 이름 처리

칩(라벨 pill)은 한 줄 고정, 토핑 우측 하단 배치.

- **그룹명 80px 초과 시 초과분부터 `…` 처리** (픽셀 기준. 구 "6자/7자 이상 → 앞 5자 + …" 문자수 기준 폐기)
- 시간 텍스트(`N분 전`)는 항상 전체 노출, 줄이지 않음
- 칩 폭은 토핑 폭에 종속시키지 않고 내용 기준으로 늘어남
- Timestamp 텍스트 컬러 = 프로필 이미지 및 토스트 닉네임 텍스트 컬러 규칙 준수 (랜덤 X)

### **상대 시간 표기 기준**

대표 토핑의 **마지막 활동 시각** 기준으로 표기.

| 경과 시간 | 표기 |
| --- | --- |
| 60초 미만 | **방금 전** |
| 1분 ~ 59분 | **N분 전** |
| 1시간 ~ 23시간 | **N시간 전** |
| 1일 ~ 6일 | **N일 전** |
| 7일 이상 | **오래 전** |

### NEW 기준

- 규칙
    - 그룹을 무조건 선택해야 읽은 것으로 판단 (C-001 진입 필요)
    - 그룹이 스크린에 나오는 것만으로 라벨에 뉴 표시가 사라지는 것이 아님
