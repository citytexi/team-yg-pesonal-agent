# 1. 배치 패턴 (지그재그)

토핑은 **행당 1개**, 좌/우를 번갈아 배치한다. 위에서 아래로 **좌 → 우 → 좌 → 우** 무한 반복.

```
[ Left ]           [ Right ] [ Left ]           [ Right ] [ Left ]    ......
```

- 소속 그룹 수가 늘어날수록 아래로 무한히 연장.
- 크림은 토핑 총량만큼 세로로 늘어나고, **체리(최상단)와 접시(최하단)는 고정 요소**.

## 1.1 시작 사이드

- 첫 번째 토핑은 **[ Left / Right ] 고정**으로 시작.

## 1.2 인덱스 기반 규칙

아이템 순서를 **0부터** 셀 때 2개 단위 반복.

- `index % 2 == 0` → **Left**
- `index % 2 == 1` → **Right**

## 1.3 저개수 특수 배치

- **N = 0**: 토핑 없음. 툴팁 노출.
- **N = 1**
- **N = 2**
- **N = 3**: 여기까지 스크롤 없음·파르페 늘어남 없음. **4부터** 파르페 템플릿 크림 개수가 늘어남.

# 2. 좌표 기준

## 2.1 측정 기준

- 모든 간격은 **토핑 프레임(160×160)** 기준.
- Img 회전·이동으로 프레임 밖으로 삐져나가는 픽셀은 측정 대상이 아님.

---

# 3. 토핑 구성 및 기본 규격

- 토핑 = **Img + Grouptag-Chip**, 6타입 (Left-1 / Left-2 / Left-3 / Right-1 / Right-2 / Right-3).
- 토핑 프레임: **160×160**
- Img: **96×96** (프레임 정중앙 기준)
- Chip: **회전 없음**
- Timestamp 텍스트 컬러 = 프로필 이미지 및 토스트 닉네임 텍스트 컬러 규칙 준수 (랜덤 X). → `raw/S-101-프로필-닉네임-컬러-규칙-v0.3.md`
- 그룹명 **80px 초과 시** 이후부터 `...` 처리.
- Z: **Img > Grouptag-Chip** (항상)
- Chip 배치 기준점: **[ 프레임 중앙 하단 / Img 중앙 하단 ]**

# 5. 그룹 수 변동 시 재계산

- **그룹 나가기** → 해당 토핑이 목록에서 빠짐.
- **빈자리 없음**: 뒤 토핑들이 정렬 순서 그대로 한 칸씩 위로 당겨짐.
- 크림은 그만큼 수축, 접시가 위로 올라옴.
- **반영 시점**: 저장·화면 재진입 시.
