## v4 → v5 변경 요약

- **담당자(어사인) 컬럼 추가**: 디자인 / iOS / AOS / Spring. (실명은 역할 코드로 마스킹, 매핑은 `wiki/personal-private/담당자-매핑.md` 참고)
- **"비고" 컬럼 삭제**, "사용자 액션" 컬럼 없음. "화면명"이 ID 자리로, 기존 화면명은 "기능명" 컬럼으로.
- **기존 A-003(진입 경로 선택 = 그룹 참여/생성 분기) 삭제** → A-003이 **"약관 동의"**로 교체. (단, A-004/A-005가 여전히 A-003을 진입경로로 참조 → 원본 흐름 불일치, 아래 공백 참고)
- **누락 화면 추가 (에러/확인/로딩 서브 화면)**:
  - C-101-confirm(촬영 확인), C-101-error(카메라 권한 없음)
  - C-102-confirm(불러온 이미지 확인), C-102-error(갤러리 권한 없음)
  - C-103-loading, C-103-select(누끼 대상 선택 — 다중 검출 시 분기)
- **캔버스 편집 화면 재번호**: C-104 "수동 누끼 편집"(신규), C-105 "수동 편집(테두리)", C-106 "캔버스에 배치"(v4의 배치+확정 통합).
- **카메라/갤러리 호출 방식**: C-101/C-102가 "커스텀 카메라/갤러리 호출"로 명시(v4 네이티브 → 커스텀).
- **프로필 이미지 결정**: A-002 기타 "프로필 이미지 넣을지 말지" → **"프로필 이미지 안넣음"**.
- **G-001 노출 정책 상세화**: 그룹 아이디/인원/참여 인원은 미노출, 그룹 이름·마지막 업로드 시간은 노출.
- **SY 명칭**: SY-001 → **SY-001-closed**(마감 갤러리 저장, 분류 Canvas), SY-003 → **SY-001-new**(신규 캔버스 자동 생성).
- **S-002 예외처리**에 "한 계정으로 여러 기기 로그인 가능 여부" 검토 항목 추가.

- **A-003 흐름 불일치**: A-003이 "약관 동의"로 바뀌었고 다음 단계는 G-001로만 표기되나, A-004(그룹 참여)·A-005(새 그룹 생성)는 여전히 진입경로로 A-003을 가리킴. "그룹 참여/생성 선택" 화면이 어디로 갔는지 원본 확인 필요.
- C-304(캔버스 배경 색 설정)는 시스템 동작 비어 있음.
- Spring 담당자 컬럼은 전 행 비어 있음(미배정).
