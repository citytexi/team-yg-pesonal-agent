# Spec: ygDimRipple — YG 커스텀 dim ripple IndicationNodeFactory

- 대상: `core:util:android` — `clickable/` (clickableYG와 동일 패키지. 2026-07-14 `core:designsystem`에서 이동)
- 관련: [ADR-0010](../adr/0010-custom-compositionlocal-theme.md)(자체 테마) · [design-system](../architecture/design-system.md) · [[2026-07-12-clickableyg-throttle|clickableYG]](이 ripple을 indication으로 받음)

## 목표
YG 자체 dim ripple을 `androidx.compose.foundation.IndicationNodeFactory`로 제공한다. `clickableYG`·`clickable`의 `indication` 인자로 끼워 은은한(subtle) 터치 리플을 낸다. 커스텀 테마(MaterialTheme 미사용)라 material3 기본 ripple 대신 색·alpha를 직접 제어한다.

## 범위
- **포함**: `ygDimRipple(...)` 팩토리 함수 + 내부 `IndicationNodeFactory`/delegating ripple `Node` + `YGDimRippleAlpha` 상수. 단일 파일.
- **제외**: `ygScaleRipple`(press 축소 효과) — **후속으로 신설됨**([[2026-07-13-clickableyg-ripple-variants]]). `ColorProducer` 동적 색 오버로드(YAGNI). ripple 색의 시맨틱 토큰화(과도기, 후속 — 현재 테마 비의존 리터럴).

## API / 인터페이스
```kotlin
@Stable
fun ygDimRipple(
    bounded: Boolean = true,
    radius: Dp = Dp.Unspecified,
    color: Color = YGDimRippleColor,
    rippleAlpha: RippleAlpha = YGDimRippleAlpha,
): IndicationNodeFactory
```
- `bounded`: 리플을 컨텐츠 경계로 클립할지. 기본 `true`.
- `radius`: 리플 반경. `Dp.Unspecified`면 컨텐츠 크기 기반 자동.
- `color`: 리플 색. 기본 `YGDimRippleColor = Color(0xFF29292C)`(어두운 회색 리터럴; dim alpha라 subtle). `core:util:android`가 테마 비의존이라 `YGAtomicColors` 대신 리터럴. 오버라이드 가능.
- `rippleAlpha`: 상태별 alpha. 기본 `YGDimRippleAlpha`.

## 동작 / 구조
- `ygDimRipple(...)`가 내부 `YGDimRippleNodeFactory`(= `IndicationNodeFactory`)를 반환.
- **`YGDimRippleNodeFactory`**: `create(interactionSource): DelegatableNode`가 `DelegatingYGDimRippleNode`를 생성. `equals`/`hashCode`를 파라미터(bounded/radius/color/rippleAlpha) 기반으로 구현 — Indication 동등성 판정에 필요.
- **`DelegatingYGDimRippleNode`**(`DelegatingNode`, `CompositionLocalConsumerModifierNode`): `onAttach`에서 `delegate(createRippleModifierNode(interactionSource, bounded, radius, color = ColorProducer { color }, rippleAlpha = { rippleAlpha }))`로 실제 리플 노드에 위임.
- **`YGDimRippleAlpha`**: `RippleAlpha(pressedAlpha = 0.15f, focusedAlpha = 0.15f, draggedAlpha = 0.15f, hoveredAlpha = 0.15f)`. 전 상태 균일 dim.
- `createRippleModifierNode`·`RippleAlpha`는 `androidx.compose.material.ripple`.

## 파일 구성 (`core:designsystem`)
- `clickable/YGDimRipple.kt` (패키지 `com.teamyg.parfait.core.util.android.clickable` — clickableYG와 동일) — public `ygDimRipple(...)` + `YGDimRippleColor`(리터럴 기본색) + `YGDimRippleAlpha` + `internal class YGDimRippleNodeFactory` + `private class DelegatingYGDimRippleNode`. (초기 `core:designsystem`의 `YGRipple.kt` 단일 파일 → `YGDimRipple.kt` 리네임 + `YGScaleRipple.kt` 분리 → 2026-07-14 `core:util:android`로 이동.)

## 주의 / 열린 질문
- **의존성**: `createRippleModifierNode`/`RippleAlpha`는 `androidx.compose.material:material-ripple`. material3 전이 포함으로 추정하되, 미해결 시 `material-ripple` 명시 의존 추가(plan compile에서 확정).
- **테마 비의존 리터럴 색**: `core:util:android`가 `core:designsystem` 비의존이라 ripple 색을 테마에서 못 읽음. 기본색을 리터럴 `YGDimRippleColor = Color(0xFF29292C)`로 둠(과거엔 `YGAtomicColors.Gray.Gray900` 참조였음). 시맨틱 토큰화하려면 호출측(designsystem 컴포넌트)이 `color`를 주입 → [open-questions](../open-questions.md).
- **테마색 미읽음**: `ygDimRipple`은 색을 파라미터로 받으므로 테마 CompositionLocal을 직접 읽지 않음(호출측이 원하는 색 전달).
- **기본 indication 주입 (해소)**: 별도 designsystem clickable wrapper 대신 [[2026-07-12-clickableyg-throttle|clickableYG]]가 같은 모듈로 이동하며 `indication` 기본값으로 `ygDimRipple()`을 직접 주입 → wrapper 후속 과제 불필요해짐.
- **네이밍**: 초기 `ygRipple`에서 `ygDimRipple`로 변경(향후 `ygScaleRipple` 등 리플 종류 확장 대비, skt `idDimRipple` 대응).
