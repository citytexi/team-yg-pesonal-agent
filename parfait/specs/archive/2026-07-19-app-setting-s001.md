## 목표
앱 사이드 메뉴(설정) 화면 S-001을 구현한다. 프로필 카드 + 리스트 4항목(계정 정보 / 서비스 이용약관 / 개인정보 처리 방침 / 버전 정보). 앞 3개는 클릭 시 해당 화면으로 네비게이트, 버전 정보는 정적 표시. **이동 타겟 화면(S-002·S-004)은 이번 범위 밖 — Route stub + entry 등록까지만** 해서 `navigator.goTo()`가 런타임 크래시 없이 동작하게 한다.

- **포함**:
  - `AppSettingScreen`(stateless UI) + `AppSettingRoute`(navigator·ViewModel 배선) + `AppSettingViewModel`/`AppSettingState`.
  - `ProfileCard` 컴포넌트 분리(`impl/component/`, login:impl 선례 `OnboardingPager`·`KakaoSignInButton`와 동일 배치).
  - 정적 UI 라벨 문자열 리소스화(`res/values/strings.xml`).
  - 이동 타겟 3종 NavKey(api) + Route stub(impl) + EntryBuilder 등록.
- **제외**:
  - 계정 정보(S-002)·서비스약관/개인정보(S-004) 화면 실제 구현 — **stub(`// TODO impl`)만**.
  - 백엔드 프로필 데이터 연동 — ViewModel placeholder emit.
  - 버전 자동 주입(`BuildConfig.VERSION_NAME`) — placeholder 문자열, 후속 TODO.
  - 리스트 항목 전체 행 클릭 — `YGListItem`(오버로드 A)는 trailing 아이콘만 클릭(Figma도 Button-Icon(44)이 탭 타겟이라 일치).

## State / ViewModel
한 파일(`AppSettingViewModel.kt`)에 State + Intent/SideEffect + ViewModel 동거(MVI, `BaseViewModel` 상속). **내비게이션은 Intent→SideEffect 경유**(Route가 아니라 VM이 결정):
```kotlin
data class AppSettingState(
    val nickname: String = "...",       // placeholder (백엔드 TODO)
    val loginProvider: String = "Kakao",
    val version: String = "1.0v",       // placeholder (BuildConfig 주입 TODO)
) : UiState

sealed interface AppSettingIntent : UiIntent {
    data object ClickBack : AppSettingIntent
    data object ClickAccount : AppSettingIntent
    data object ClickServiceTerms : AppSettingIntent
    data object ClickPrivacyPolicy : AppSettingIntent
}

sealed interface AppSettingSideEffect : UiSideEffect {
    data object NavigateBack : AppSettingSideEffect
    data object NavigateToAccountInfo : AppSettingSideEffect
    data object NavigateToServiceTerms : AppSettingSideEffect
    data object NavigateToPrivacyPolicy : AppSettingSideEffect
}

@HiltViewModel
class AppSettingViewModel @Inject constructor() :
    BaseViewModel<AppSettingState, AppSettingIntent, AppSettingSideEffect>(
        initialState = AppSettingState(),
    ) {
    // processIntent → intent별 private handle* 함수 위임 → 각 handle*가 postSideEffect 호출
    override fun processIntent(intent: AppSettingIntent) = when (intent) {
        AppSettingIntent.ClickBack -> handleClickBack()
        AppSettingIntent.ClickAccount -> handleClickAccount()
        AppSettingIntent.ClickServiceTerms -> handleClickServiceTerms()
        AppSettingIntent.ClickPrivacyPolicy -> handleClickPrivacyPolicy()
    }

    private fun handleClickBack() = postSideEffect(AppSettingSideEffect.NavigateBack)
    // handleClickAccount/ServiceTerms/PrivacyPolicy → 각 Navigate* SideEffect postSideEffect
}
```
- 런타임 상태: 정적(백엔드 연동 전). `state: StateFlow<AppSettingState>`(BaseViewModel)로 placeholder 노출.
- 각 intent는 **private `handleClick*()` 함수로 랩핑**한 뒤 `postSideEffect(...)` 호출(when 분기가 postSideEffect를 직접 호출하지 않음).
- 동적 값(nickname/loginProvider/version)은 **State 소유(데이터)** — 문자열 리소스 대상 아님.

## 네비게이션 배선 (MVI Intent→SideEffect)
- 흐름: `AppSettingScreen` 콜백 → `viewModel.processIntent(AppSettingIntent.*)` → VM `handleClick*()` → `postSideEffect(AppSettingSideEffect.*)` → `AppSettingRoute`가 `LaunchedEffect`로 `viewModel.effect` 수집 → `navigator` 호출.
- `AppSettingRoute` 콜백 배선(Screen→Intent):
  - `onClickBack = { viewModel.processIntent(AppSettingIntent.ClickBack) }`
  - `onClickAccount = { viewModel.processIntent(AppSettingIntent.ClickAccount) }`
  - `onClickTerms = { viewModel.processIntent(AppSettingIntent.ClickServiceTerms) }`
  - `onClickPrivacy = { viewModel.processIntent(AppSettingIntent.ClickPrivacyPolicy) }`
- `AppSettingRoute` effect 수집(SideEffect→navigator):
  - `NavigateBack → navigator.onBack()`
  - `NavigateToAccountInfo → navigator.goTo(NavKeyAccountInfo)`
  - `NavigateToServiceTerms → navigator.goTo(NavKeyServiceTerms)`
  - `NavigateToPrivacyPolicy → navigator.goTo(NavKeyPrivacyPolicy)`
- `EntryBuilder`(`featureAppSettingEntryBuilder`)에 entry 추가:
  - `entry<NavKeyAppSetting>` → `AppSettingRoute` (기존). Scaffold 배경 `containerColor = YGAtomicColors.Gray.White`(S-001 화면 흰 배경).
  - `entry<NavKeyAccountInfo>` → `AccountInfoRoute` (stub)
  - `entry<NavKeyServiceTerms>` → `TermsRoute` (stub)
  - `entry<NavKeyPrivacyPolicy>` → `TermsRoute` (stub, 공용)
- `Navigator` API: `goTo(NavKey)` / `onBack()` (`core:navigation`).

## 검증
- **자동화 테스트 없음** — 이 프로젝트는 현재 테스트 코드 선례가 없다(테스트 미도입). 별도 지침 전까지 테스트 파일을 만들지 않는다.
- 검증선: `:feature:app:setting:impl` **컴파일**(`compileDebugKotlin`) + **ktlint** 통과.
- `AppSettingScreen`·`ProfileCard`: `@YGPreview`+`PreviewBox` 프리뷰로 Android Studio 육안 확인.
- nav 동작: 3개 항목 클릭 → `goTo` → stub 화면(`// TODO impl`) 진입, back 복귀(런타임 크래시 없음) 수동 확인.
