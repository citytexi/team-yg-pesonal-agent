# Spec: YGAlert 배너 알림 + 노출 정책 (YGAlertHost)

> 상태·날짜·대상·관련은 frontmatter가 단일 출처. 본문은 설계에 집중.

- 관련: [ADR-0010](../../adr/0010-custom-compositionlocal-theme.md)(자체 테마) · [design-system](../../architecture/design-system.md)

## 목표
상단에서 내려오는 배너형 알림. **표시(YGAlert)** 와 **노출 정책(YGAlertPolicy/YGAlertHost)** 을 분리해,
화면은 정책 홀더에 `show()`만 하고 자동 소멸·스와이프 닫기·애니메이션은 Host가 담당한다.
(YGToast와 동일한 Policy/Host 패턴 — [ygtoast](2026-07-23-ygtoast.md) 참조.)

## 범위
- 포함: 배너 컴포저블, 단일 슬롯 정책 홀더, Host(자동 소멸·위로 스와이프 닫기·슬라이드 인/아웃).
- 제외: 문자열 생성·다국어 매핑(호출자 소유). 완성 문자열만 받는다.

## API / 인터페이스
```kotlin
@Composable
fun YGAlert(
    title: String,
    sub: String,
    modifier: Modifier = Modifier,
    buttonText: String? = null,
    onButtonClick: (() -> Unit)? = null,
)

data class YGAlertItem(val id: String, val title: String, val sub: String, val visible: Boolean = true)

@Stable
class YGAlertPolicy {
    var alert: YGAlertItem?           // 단일 슬롯(스택 아님)
    fun show(title: String, sub: String)
    internal fun clearAlert()
}

@Composable fun rememberYGAlertPolicy(): YGAlertPolicy
@Composable fun YGAlertHost(policy: YGAlertPolicy, modifier: Modifier = Modifier)
```

## 레이아웃 / 토큰 매핑 (심볼명)
- 루트 `Row`(수직 중앙) — `fillMaxWidth`, 배경 `YGAtomicColors.Transparency.Black75`, 패딩 세로 `YGTheme.layout.padding.padding5` / 가로 `padding7`.
- 좌측 텍스트 `Column`(`weight(1f)`, `gap.gap2`):
  - `title`: `YGTheme.typography.body.b02SB` · `YGAtomicColors.Cherry.Cherry200`.
  - `sub`: `body.b02R` · `YGAtomicColors.Transparency.White75`.
- `buttonText != null`이면 우측에 `YGChipButton`(`YGChipButtonColorsDefaults.CherryBackgroundPressed`, `endIconResource = ic_caret_right`, `onClick = onButtonClick ?: {}`).

## 노출 정책 (YGAlertPolicy / YGAlertHost)
- **단일 슬롯**: `alert`(nullable) 하나만 보유 — 새 `show()`가 이전 배너를 대체(스택 없음. YGToast와 대비).
- **자동 소멸**: `YGAlertHost`가 `LaunchedEffect(alert.id)`에서 `delay(DISMISS_DELAY)` 후 `clearAlert()`.
- **애니메이션**: `AnimatedVisibility` + `slideInVertically/slideOutVertically`(위에서 진입·이탈, `tween(ANIMATION_DURATION)`).
- **위로 스와이프 닫기**: `draggable`(Vertical) — 음수 델타만 누적, `onDragStopped`에서 오프셋이 `SWIPE_DISMISS_THRESHOLD` 미만이거나 속도가 `FLING_DISMISS_VELOCITY` 미만이면 `clearAlert()`, 아니면 오프셋 리셋.
- 상수: `DISMISS_DELAY` · `ANIMATION_DURATION` · `SWIPE_DISMISS_THRESHOLD` · `FLING_DISMISS_VELOCITY`(파일 private const, 값은 코드 SoT — 파르페 규율상 미기재).
