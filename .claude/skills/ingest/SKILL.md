# ingest — 소스 흡수 워크플로우

`raw/`의 소스를 읽어 `wiki/`에 sources/entities/concepts 페이지를 생성·갱신한다.
세부 규칙(페이지 생성 임계 기준, frontmatter, Obsidian 링크, 도메인 태그)은 항상 `wiki/CLAUDE.md`(위키 스키마)를 따른다.

## 모드 판단

- **단일/인터랙티브**: 소스 1개를 줄 때. 2단계(논의)를 수행한다.
- **배치**: "배치로", "한꺼번에", 여러 소스를 동시에 줄 때. 2단계(논의)를 생략하고 끝에 요약 보고서를 한 번 제공한다.

## 단계

1. **읽기**: 대상 파일을 읽는다. 이미지가 있으면 텍스트를 먼저 처리하고, 데이터·도표·텍스트가 담긴 이미지만 이후 별도로 본다. 단순 장식·배너는 건너뛴다. 텍스트 전용 소스면 이미지 단계 생략.
2. **논의** (단일 모드만): 핵심 내용·발견 엔티티·개념을 사용자와 간략히 논의한다.
3. **sources/ 페이지 생성**: `wiki/sources/[파일명].md`에 요약 페이지를 `wiki/CLAUDE.md`의 템플릿대로 작성한다.
4. **entities/ 업데이트**: 페이지 생성 임계 기준(2개 이상 소스에 등장 OR 단독 재참조 가능성 높음)을 만족하는 엔티티만 생성/병합. 모순 발견 시 해당 섹션에 `> ⚠️ [날짜] [[원본파일]] 과 상충` 마커를 추가하고 `open-questions.md`에 항목 등록.
5. **concepts/ 업데이트**: 임계 기준을 만족하는 개념만 신규 생성/업데이트.
6. **overview.md 갱신**: 이 소스가 현재 논지를 강화·반박하면 `wiki/overview.md`를 갱신.
7. **index.md 업데이트**: 새 페이지를 카탈로그에 추가.
8. **log.md 추가**: 파일 끝에 `## [YYYY-MM-DD] ingest | [원본 제목]` 한 줄 기록.

## 배치 완료 보고

전체 완료 후: "N개 소스 처리, M개 페이지 생성/업데이트, K건 모순 발견" 형식으로 한 번에 보고. 모순·불명확한 판단이 필요할 때만 중간에 사용자에게 확인.

## 주의

- `raw/`는 읽기 전용. 절대 수정·생성·삭제하지 않는다.
- 내부 링크는 `[[파일명]]` 형식만 사용.
