# lint — 위키 상태 점검 워크플로우

`wiki/` 전체를 점검해 문제를 보고한다. 세부 규칙은 항상 `wiki/CLAUDE.md`(위키 스키마)를 따른다.

## 점검 항목

1. **민감 데이터**: 이메일·전화번호·API 키·비밀번호 패턴 탐지. **발견 시 자동 수정 금지** — 파일명:줄번호를 사용자에게 보고하고 처리 방법(삭제·마스킹·private 폴더 이동)을 확인한다. 처리 전까지 커밋 금지.
2. **모순 감지**: 같은 주제 페이지 간 상충 내용 → `open-questions.md`에 미해결 항목 등록.
3. **고아 페이지**: 어떤 페이지에서도 `[[링크]]`되지 않은 `wiki/` 파일.
4. **링크 없는 언급**: 페이지 없이 언급만 되는 엔티티·개념. 페이지 생성 임계 기준 충족 시 생성 권고, 미달이면 무시.
5. **데이터 공백**: 채울 수 있는 빈 영역 또는 조사 권고 주제.
6. **stale 정보**: 최근 원본과 모순되는 오래된 내용.

## 출력

`wiki/synthesis/lint-YYYY-MM-DD.md`에 `wiki/CLAUDE.md`의 보고서 템플릿대로 저장한다.
완료 후 `log.md` 끝에 `## [YYYY-MM-DD] lint | 점검 완료, N건 발견` 기록.
